## 设计说明

### 核心设计理念

#### 1. 资源绑定模式
- **拼车组模式（一对一绑定）**:
  - 创建组时选择"一对一绑定"
  - 整个拼车组绑定唯一的AI账号
  - 所有API密钥自动使用该账号
  - 简化管理，确保资源公平使用

- **企业模式（账号池）**:
  - 企业级账号池管理
  - API密钥可智能分配或指定账号
  - 支持负载均衡和故障转移
  - 适合大规模团队使用

#### 2. AI账号配置
- **Claude系列**:
  - OAuth模式：Claude Code直连
  - Console模式：支持原生API和多种中转服务
  - 中转服务：Kimi k2、通义千问、智谱GLM、自定义

- **其他CLI工具**:
  - Gemini CLI、Cursor Agent、AmpCode CLI、Auggie CLI
  - 统一的API Key格式和配置方式

### 布局原则
1. **清晰的信息层级**: 重要信息优先展示
2. **功能分组**: 相关功能集中放置
3. **视觉引导**: 使用分隔线和空白区分模块
4. **响应式设计**: 适配不同屏幕尺寸
5. **模式区分**: 明确区分拼车组和企业模式的不同功能

### 交互元素
- `[按钮]`: 可点击的操作按钮
- `[✓]`: 复选框（已选中）
- `[ ]`: 复选框（未选中）
- `●/○`: 单选按钮
- `▼`: 下拉菜单指示器
- `[输入框]`: 文本输入区域
- `📊📈🔑👥⚙️🔗`: 功能图标
- `⚠️ ℹ️`: 提示和警告信息

### 状态指示
- `●`: 在线/活跃状态（绿色）
- `○`: 离线/禁用状态（灰色）
- `⚠️`: 警告状态（黄色）
- `❌`: 错误状态（红色）
- 进度条: `████████░░` 表示使用率

### 响应式断点
- 手机: < 576px
- 平板: 576px - 1024px
- 桌面: > 1024px

### 页面结构说明
1. **拼车组创建**: 包含资源绑定模式选择
2. **AI资源配置**: 拼车组模式下只能绑定一个账号
3. **API密钥管理**: 拼车组模式无需选择账号，企业模式支持智能分配
4. **CLI配置指导**: 统一的环境变量配置方式
5. **组织仪表盘**: 展示绑定账号状态和使用情况

这些ASCII原型图展示了AiCarpool系统的核心页面设计，涵盖了从组织创建、AI资源配置到API密钥管理的完整流程，特别强调了拼车组模式和企业模式的差异化设计。
